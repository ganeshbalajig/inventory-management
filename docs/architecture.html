Data Flow

1
User changes a filter
The FilterBar.vue component binds to filter refs in the useFilters composable — a module-level singleton, so state is shared across all views without a global store.
2
View reacts to filter change
Each view watches the filter refs. When any value changes, the view calls loadData(), sets a loading state, and invokes the relevant api.js method.
3
api.js builds query string
api.js calls getCurrentFilters(), skips any filter set to 'all', and appends the rest as URLSearchParams before firing the Axios GET request to http://localhost:8001/api/....
4
FastAPI applies filters in-memory
The endpoint calls apply_filters() for warehouse, category, and status, then filter_by_month() for time period. Month values can be a direct YYYY-MM string or a quarter like Q1-2025, which expands to three months via QUARTER_MAP.
5
Pydantic validates and serializes
FastAPI runs each result dict through the declared response_model (e.g., List[Order]), validating types and stripping undeclared fields before serializing to JSON.
6
Vue updates derived state via computed properties
The raw response lands in a ref (e.g., allOrders). Downstream computed properties — totals, filtered lists, chart data — recalculate automatically. The template re-renders only the affected parts.

Filter System

Time Period Warehouse Category Order Status
Endpoint Time Period Warehouse Category Status
/api/orders ✓ ✓ ✓ ✓
/api/dashboard/summary ✓ ✓ ✓ ✓
/api/inventory — ✓ ✓ —
/api/demand — — — —
/api/backlog — — — —
/api/spending/* — — — —

Time Period accepts YYYY-MM (direct month match) or Q1-2025 … Q4-2025 (expands to three months via QUARTER_MAP in backend). All filters default to 'all' — the 'all' value is explicitly skipped so the filter has no effect.

Data Files — server/data/

inventory.json
SKU, name, warehouse, quantity, reorder points, unit cost
orders.json
Customer orders with status, line items, delivery dates
backlog_items.json
Delayed order items with priority and quantity shortfall
demand_forecasts.json
Per-SKU current vs forecasted demand with trend direction
purchase_orders.json
Supplier POs linked to backlog items by backlog_item_id
spending.json
Summary KPIs — YTD spend, budget, savings
transactions.json
Recent purchase transactions with supplier and category

All files are loaded once at server startup via mock_data.py into Python module-level lists. Changes to JSON files require a server restart to take effect. Data does not persist across restarts — the lists are read-only in memory.
